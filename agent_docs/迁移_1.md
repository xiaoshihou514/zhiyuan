## 一、目标状态定义

迁移完成后，致远应呈现以下特征：

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| 核心能力 | 信息收集器 | 知识建构系统 |
| 匹配方式 | 关键词为主 | 语义为主，关键词兜底 |
| 评分逻辑 | 统计指标（数量、密度） | 结构指标（论证完整度、解释层级） |
| 终止信号 | 信息饱和（novel_count < 1） | 认知饱和（解释框架稳定） |
| 长报告 | 章节平铺（厚但浅） | 论证递进（薄但深） |
| 研究策略 | 固定并行搜索 | 自适应分层深挖 |
| 可信度 | 来源数量统计 | 来源网络拓扑分析 |

---

## 二、架构新增与改造

### 新增基础设施层

```
EmbeddingProvider（语义编码服务）
├── 本地轻量模型（默认，bge-m3 或同级）

VectorIndex（向量索引）
├── HNSW 内存索引（实时 ANN 检索）
├── 跨研究持久化（序列化/反序列化）
└── 与 RocksDB 协同（原始数据存 RocksDB，向量存索引）
```

### 改造现有组件

| 组件 | 改造内容 |
|------|---------|
| **SemanticMemory** | 从关键词遍历升级为 ANN 向量检索 |
| **QualityEvaluator** | 四维指标全部基于语义结构和论证网络 |
| **ExtractorAgent** | 相关性判断从关键词匹配改为 embedding 相似度 |
| **SynthesizerAgent** | 输出增加 `epistemic_status`（对论证骨架的影响类型） |
| **WriterAgent** | 从"按主题归类"写作改为"按论证递进"写作 |
| **Orchestrator** | 终止条件改为解释框架稳定度 |

---

## 三、评分系统重构（P0）

### 新四维指标

| 维度 | 定义 | 评估方式 |
|------|------|---------|
| **覆盖度** | 论证骨架的完整度 | 核心论点是否具备"前提-证据-推论"三层结构 |
| **可信度** | 来源网络的独立性和溯源深度 | 信息是否追溯到原始出处，有效来源数/表面来源数 |
| **深度** | 因果解释层级 | 从描述性（是什么）→ 因果性（为什么）→ 反事实（如果会怎样）的分布 |
| **新颖度** | 解释框架的演进幅度 | 新增发现是否迫使修正现有论点，而非仅补充细节 |

### 关键变化

- 引入**论证骨架**作为中间层：findings 不直接评分，先映射到论证骨架中的角色，再评分
- 评分不再基于 findings 的静态属性，而是基于 findings 在**认知结构中的位置**

---

## 四、终止条件重构（P1）

### 旧逻辑
> max_iterations 或连续 2 轮 novel_count < 1

### 新逻辑
> 当"解释框架稳定度"连续两轮超过阈值时终止

**稳定度信号**（三选一触发）：
- 连续两轮新增 finding 的 80% 以上被 SynthesizerAgent 标记为"补充细节"
- `extract_directions` 返回的方向 priority 均值 < 0.3
- WriterAgent 判断报告更新"无需新增或合并章节"

### 配套机制

- **允许超挖**：若某核心论点可信度不足，即使框架稳定也触发定向深挖
- **早停标记**：子任务级别的解释框架稳定后，标记完成不再搜索

---

## 五、语义匹配全面替换（P1-P2）

### 替换矩阵

| 原有关键词匹配 | 替换为 | 组件 |
|---------------|--------|------|
| 子任务 vs findings 关键词命中 | embedding 余弦相似度 | QualityEvaluator |
| findings Jaccard 去重（>0.5） | embedding 余弦相似度（>0.85） | Orchestrator |
| 搜索结果标题摘要关键词匹配 | 搜索结果 vs 任务描述的 embedding 相似度排序 | ExtractorAgent |
| 语义记忆遍历关键词匹配 | ANN 近似最近邻搜索 | SemanticMemory |
| 子任务覆盖判断 | 语义聚类 + 概念空白检测 | SynthesizerAgent |

### 降级策略

embedding 服务不可用时，自动回退到关键词匹配，保证系统可用性。

## 六、长报告结构重构（P2-P3）

### 旧流水线
```
大纲 → 分配 finding → 按章写作 → 校对重复/矛盾
```

### 新流水线
```
生成论证骨架（核心论点 + 预期推理链 + 反事实检验点）
    ↓
迭代填充证据（每轮评估：支持/削弱/无关/延伸 哪个论点）
    ↓
按"论点推进"组织章节（非 finding 归类）
    ↓
交叉校对：论证闭环性 + 反事实覆盖 + 来源独立性
```

### PlannerAgent 输出扩展

增加 `core_thesis` 和 `expected_reasoning_chain`，大纲不仅是章节列表，更是**待验证的假设体系**。

### SynthesizerAgent 输出扩展

增加 `epistemic_status`：
- `supports`：强化现有论点
- `undermines`：削弱现有论点（触发修正）
- `extends`：扩展论点边界（可能新增子论点）
- `irrelevant`：与当前论证骨架无关（暂存或丢弃）

### WriterAgent 写作逻辑

章节结构由"主题分类"变为"论证递进"：
- 每章回答一个**为什么**或**那又怎样**
- 章节间存在逻辑依赖（后章建立在前章结论上）
- 交叉校对检查"删除某章是否导致论证断裂"

---

## 七、搜索策略自适应分层（P2-P3）

### 四层深挖模型

| 层级 | 目标 | 触发条件 | 查询特征 |
|------|------|---------|---------|
| **侦察层** | 建立领域地图 | 每研究必执行 | 宽泛查询，general 类别，识别关键概念和争议点 |
| **证据层** | 收集支撑/反驳材料 | 论点缺乏数据支撑 | 定向查询，science/news 组合，锁定具体数据 |
| **机制层** | 深挖因果和反事实 | 论点缺乏解释深度 | 追加 mechanism、limitations、failed、comparison |
| **溯源层** | 验证关键数据出处 | 关键数据单一来源 | 以数据点为查询，追踪原始论文/财报/官方文档 |

### 动态控制

- 非每层必跑，由**当前论证骨架的缺口**自动触发
- 侦察层只读摘要和目录，不提取全文
- 后续层根据缺口定向执行，避免广撒网

## 八、可信度体系网络重构（P3-P4）

### 从二部图到来源网络

```
构建来源引用图（source A 引用/链接到 source B）
    ↓
识别信息源头（入度最高的节点 = 原始出处）
    ↓
标记回声室（多个来源共享同一上游）
    ↓
计算来源独立性 = 去重后的有效来源数 / 表面来源数
    ↓
可信度 = f(溯源深度, 来源独立性, 原始出处权威性)
```

### ExtractorAgent 扩展

除提取正文外，提取网页中的**超链接和参考文献列表**，为来源网络提供边。

### VerifierAgent 改造

从"逐 claim-source 判断 supports/contradicts"改为：
- **规则层**：同一域名/同一上游的多个页面视为单源
- **向量层**：claim 与 source 语义相似度 < 0.2 直接跳过
- **拓扑层**：已被高可信度节点支持的 claim 不再重复验证
- **LLM 层**：只处理"高争议"（多源矛盾）和"高影响"（支撑核心论点）的 claim-source 对

## 九、用量控制机制（贯穿全程）

### 核心原则

> 不是"少调用 LLM"，而是"让 LLM 在关键节点做关键判断"

### 具体策略

| 策略 | 实现 | 效果 |
|------|------|------|
| **分层过滤** | 规则/向量预筛 → 粗聚类 → LLM 精筛 | 80% 内容在 LLM 前被过滤 |
| **提取截断动态化** | 非关键 URL 截断 500 字符，关键 URL 给 3000+ | 节省 60-70% 提取 token |
| **论证骨架增量更新** | 维护持久化骨架，每轮只局部修改 | 减少 writer 输入 80% |
| **异步聚合验证** | 不每轮验证，攒 2-3 轮后批量验证 | 减少 60% verifier 调用 |
| **缓存论证判断** | 相同 claim+source 组合复用历史 verdict | 跨轮/跨研究节省 30-50% |
| **早停标记** | 子任务解释框架稳定后不再搜索 | 减少 20-40% 搜索量 |

### 用量预期

| 模式 | 当前用量 | 迁移后用量 | 质量变化 |
|------|---------|-----------|---------|
| 短报告 | 基准 | +10-20% | 深度 ↑ 200% |
| 长报告 | 基准（200-300 网页） | +30-50% | 深度 ↑ 300%，从"厚而浅"到"深而精" |

## 十、实施优先级与里程碑

| 阶段 | 周期 | 内容 | 验证标准 |
|------|------|------|---------|
| **P0** | 2 周 | EmbeddingProvider + SemanticMemory 向量检索 | 跨研究知识复用能召回语义相关但词汇不同的历史发现 |
| **P1** | 2 周 | 评分系统重构（论证骨架 + 新四维）+ 终止条件改为解释框架稳定度 | 同一测试查询，新旧评分差异 > 30%，终止轮次更合理 |
| **P2** | 3 周 | SynthesizerAgent 增加 epistemic_status + 搜索策略自适应分层 | 长报告的研究方向识别从"词汇空白"变为"概念空白" |
| **P3** | 3 周 | WriterAgent 按论证递进写作 + 长报告交叉校对升级 | 长报告阅读测试：读者能清晰复述核心论点及其推理链 |
| **P4** | 2 周 | 来源网络分析 + VerifierAgent 改造 | 能自动识别并标记回声室和孤立来源 |
| **P5** | 持续 | 全量语义匹配替换 + 性能调优 | 关键词匹配仅作为降级 fallback |

## 十一、风险与应对

| 风险 | 应对 |
|------|------|
| 本地 embedding 模型质量不足 | 默认 bge-m3（多语言 SOTA 级），预留远程 API 接口 |
| HNSW 索引内存膨胀 | 百万级向量 < 1GB，超限时分片或迁移 Qdrant |
| 论证骨架设计过度复杂 | 先跑通"核心论点 + 支撑/削弱"两层，再扩展 |
| 长报告写作逻辑跳跃 | 强制章节间插入"过渡句"检查点，LLM 自评连贯性 |
| 用户不适应新评分 | 保留旧评分展示为"参考指标"，新评分标注为"结构质量" |
