# 研究流水线（Research Pipeline）

## 总体流程

```
用户查询
    │
    ▼
┌────────────────────────────────┐
│ 阶段一：意图澄清                │  ← PlannerAgent 生成 2-4 个问题
│  （--clarify 默认开启）         │     TUI 交互收集用户反馈（可跳过）
└──────────────┬─────────────────┘
               │
               ▼
┌────────────────────────────────┐
│ 阶段二：研究规划                │  ← PlannerAgent.create_plan()
│  短报告：3-6 子任务             │     长报告：3-8 章大纲 + 每章子任务
└──────────────┬─────────────────┘
               │
               ▼
    ┌─── 迭代循环（1~N 轮）───┐
    │                         │
    │  ┌─────────────────┐    │
    │  │ 1. 构建任务列表   │    │  ← pending 子任务 + 待探索方向（优先级≥0.5）
    │  └────────┬────────┘    │
    │           ▼             │
    │  ┌─────────────────┐    │
    │  │ 2. 并发执行任务   │    │  ← 每任务内部：查询→搜索→提取→综合→(交叉验证)
    │  │   (Semaphore)    │    │     按 config.concurrency 分块
    │  └────────┬────────┘    │
    │           ▼             │
    │  ┌─────────────────┐    │
    │  │ 3. 合并相似发现   │    │  ← merge_findings()：Jaccard 相似度 >0.5 去重
    │  └────────┬────────┘    │     返回 novel_count（新增新颖发现数）
    │           ▼             │
    │  ┌─────────────────┐    │
    │  │ 4. 验证引用关系   │    │  ← VerifierAgent: 声明↔来源 支持/矛盾
    │  └────────┬────────┘    │
    │           ▼             │
    │  ┌─────────────────┐    │
    │  │ 5. 质量评估      │    │  ← QualityEvaluator: 4 维评分
    │  └────────┬────────┘    │
    │           ▼             │
    │  ┌─────────────────┐    │
    │  │ 6. 提取新方向     │    │  ← SynthesizerAgent.extract_directions()
    │  └────────┬────────┘    │
    │           ▼             │
    │  ┌─────────────────┐    │
    │  │ 7. 更新报告草稿   │    │  ← WriterAgent.update_report()
    │  └────────┬────────┘    │
    │           │             │
    └───────────┼─────────────┘
                │ 终止条件判断
                ▼
┌────────────────────────────────┐
│ 阶段三：最终报告                │
│  短报告：WriterAgent 最终更新   │
│  长报告：按章节划分→交叉校对→   │
│          WriterAgent 编译       │
└──────────────┬─────────────────┘
               │
               ▼
┌────────────────────────────────┐
│ 阶段四：PDF 生成（可选）        │
│  Typst 编译 → 错误修复(最多5轮) │
└────────────────────────────────┘
```

## 单任务内部流程 (`process_single_task`)

每个子任务独立执行以下子步骤：

```
任务描述
    │
    ▼
┌──────────────────────┐
│ QueryPlannerAgent    │  → 生成 2-4 个搜索查询 + SearXNG 分类
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│ SearcherAgent        │  → 并发执行搜索
│  normal: search()    │     (fallback 顺序)
│  cross-validate:     │     (所有引擎并发 + 去重排序)
│    search_all()      │
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│ ExtractorAgent       │  → 域名过滤 + 片段匹配 → top-10 提取
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│ SynthesizerAgent     │  → 新内容 vs 已有发现比对 → 新 finding
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│ 交叉验证（可选）      │  → LLM 逐条判断 TRUE/FALSE
│ (cross_validate)     │     相似度≥0.3 跳过 LLM 直接保留
└──────────┬───────────┘
           ▼
     存储到 episodic memory
```

## 迭代控制

| 条件 | 行为 |
|------|------|
| 达到 `max_iterations`（默认 4） | 终止 |
| 连续 2 轮 `novel_count < 1` | 提前终止（平台期） |
| 研究进行中 | 每轮都执行完整流程 |

注意：**不再使用质量阈值判定终止**。质量评分仅供用户参考和报告展示。

## 长报告模式（`--long`）

1. `PlannerAgent::create_long_plan()` 生成多章节大纲 + 按章节归类的子任务
2. 迭代循环与短报告相同，但 `max_iterations` 限制为 `min(3, config.max_iterations)`
3. 迭代结束后额外步骤：
   - `assign_findings_to_chapters()` — LLM 将 findings 分配到对应章节
   - `cross_check_chapters()` — LLM 检查章节间重复/矛盾/遗漏
   - `WriterAgent::write_long_report()` — 编译完整长报告

## 交叉验证模式（`--cross-validate`）

- `--long` 模式下自动开启，或配置 `[research].cross_validate = true`
- 搜索阶段：`EnginePool.search_all()` 并发所有引擎，结果按跨引擎覆盖数排序
- 综合阶段：LLM 逐条事实核查（text_similarity ≥0.3 跳过 LLM 节省 token）
- 被过滤的发现会被记录在日志中

## 渐进式报告构建

- 每轮迭代结束后调用 `WriterAgent::update_report()` 更新报告草稿
- 下一轮以当前草稿 + 新发现作为输入
- 最终输出：短报告为单节 + 更新；长报告为多章节完整报告
